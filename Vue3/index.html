Vue3笔记

setup语法糖

  • setup函数是组合式API的入口函数,是 Vue3 特有的选项，作为组合式API的起点
  • 从组件生命周期看，它在 beforeCreate 之前执行
  • 函数中 this 不是组件实例，是 undefined
<script setup>
import { ref } from 'vue';

const message = ref('Hello, World!');

function increment() {
  message.value += '!';
}
</script>

<!-- vue3模板中可以有多个根标签 -->
<template>
  <div>
    <h1>{{ message }}</h1>
    <button @click="increment">增加</button>
  </div>
</template>

响应式

reactive

  • 在vue2中,data中定义的数据是响应式的
  • 在vue3中需要引入响应式方法,定义响应式数据
  • reactive 函数通常定义复杂类型的响应式数据,不可定义简单数据类型
<script setup>
//引入响应式方法    
import { reactive } from 'vue';

const data = reactive({
  msg: 'Hello, World!'
});

function changeMsg() {
  data.msg = '改变了msg';
}
</script>

<template>
  <div>
    <h1>{{ data.msg }}</h1>
    <button @click="changeMsg">改变消息</button>
  </div>
</template>

ref

  • 使用 ref 创建的数据，js 中需要 .value ，template 中可省略
  • 可以定义简单数据类型和复杂数据类型
<script setup>
//导入函数
import { ref } from "vue";
//创建响应式数据对象
const count = ref(0);
const increment = () => {
  // js中使用需要.value获取其值
  count.value += 10;
  // 使用ref定义复杂数据类型
  const obj = ref({
    age:18
  })
  console.log(obj.value.age) // 18
</script>

<template>
  <div>
    <p>
      计数器：{{ count }}
      <button @click="count++">累加1</button>
      <!-- template中使用可省略.value -->
      <button @click="increment">累加10</button>
    </p>
  </div>
</template>
  • 其它定义响应式数据的方法 : shallowReactive 只定义对象最外一层的响应式 shallowRef 等等

响应式原理

  • Proxy 实现：Vue 3 使用 JavaScript 的 Proxy 对象配合Reflect来实现响应式系统，而不再使用 Vue 2 中的 Object.defineProperty。Proxy 可以捕获更多类型的操作，并且可以轻松地追踪对象的新增和删除属性,解决了Vue2中响应式的不足

computed与watch

<script setup>
  //computed与vue2使用几乎一致
  import { ref, computed } from "vue";

  const list = ref([80, 100, 90, 70, 60]);
  // 计算属性
  const filterList = computed(() => list.value.filter((item) => item >= 90));
  // 改变数据，计算属性改变
  setTimeout(() => {
    list.value.push(92, 66);
  }, 3000);

</script>

<template>
  <div>
    <p>分数：{{ list }}</p>
    <p>优秀：{{ filterList }}</p>
  </div>
</template>

watch语法

  • watch与vue2中有所变化,可以监听单个或多个
  • watchEffect 智能监视 跟计算属性相似 不用指定监视谁,用到谁就会监视谁
  • 计算属性更注重返回值 而这个监视更看重流程 过程
// watch监听单个数据变化
const wc = ref(666)
watch(wc, (newval, oldval) => {
  console.log(newval, oldval);
})
setTimeout(() => wc.value = 888, 2000)

// watch监听多个数据变化
const wc1 = ref(666)
const res1 = ref(999)
watch([wc1, res1], (newval, oldval) => {
  console.log(newval);
  console.log(oldval);
})
setTimeout(() => {
  wc1.value = 999
  res1.value = 666
}, 2000)

//watch监听对象属性变化
const obj = ref({name:'wc'})
watch(()=>obj.value.name,()=>{
    .....
},{deep:true})

class Person {
  // 构成函数
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  // 示例方法
  greet() {
    console.log(`Hello, my name is ${this.name} and I'm ${this.age} years old.`);
  }

  // 示例方法
  celebrateBirthday() {
    this.age++;
    console.log(`Happy birthday! Now I'm ${this.age} years old.`);
  }

  // 静态方法
  static getVotingAge() {
    return 18;
  }
}

// 创建 Person 对象的实例
const person = new Person('John Doe', 30);

// 访问属性
console.log(person.name);
console.log(person.age);

// 调用示例方法
person.greet();
person.celebrateBirthday();

// 调用静态方法
const votingAge = Person.getVotingAge();
console.log(`The voting age is ${votingAge}.`);

//子类可以使用父类的方法,通过super关键字
//子类可以重写父类的方法
//修饰符  readonly只读属性   static为设置静态属性/方法  私有方法简写#  具体看文档

const props = withDefaults(defineProps<{

})

const wc = () => {
  emit('changeMoney',1)
}
