Vue2笔记

Vue基础指令

v-bind

  • v-bind 用来绑定标签属性,简写形式为 : 示例: :value 或 :src

v-on

  • v-on 用来绑定事件 简写形式 : @事件名='vue函数方法'

v-once

  • 初次渲染后就为静态内容了,以后变量更新不在渲染 这个命令没有值

v-text与v-html

  • 两者都能往标签里添加内容,都会覆盖插值表达式,v-text不能解析标签,v-htm可以解析标签
  • v-html存在安全性问题,永远不要把v-html用在用户提交的内容上

事件对象

  • $event 在标签里的函数里写,为事件对象,也能当参数 这样如果一个函数需要传入参数时,事件对象不会丢失

v-model

  • 可以实现数据双向绑定,vue的data里的属性可以获取到表单值
<template>
<!-- v-model 数据双向绑定  -->
  <form action="">
    姓名:<input type="text" v-model="text"><br>
    密码:<input type="password" v-model="pass"><br>

    <input type="radio" value="男" v-model="gender" name="sex">男

模板语法

  • 插值表达式; 使用双大括号{{ }}将表达式插入到模板中。在模板渲染时，表达式将被实际的数据值替换
  • vue2模板中,只允许有一个根标签,vue3可以有多个

v-show与v-if

  • v-show 用得是display:none隐藏,频繁切换时使用
  • v-if 是从dom树直接删除与显示,少量切换时使用
  • 两者变量为true,则显示v-show='true', false则隐藏
  • v-if可以配合v-else,v-else-if 使用,与if else用法一样,后面跟条件,true为显示 例:v-if='age>18'

v-for与更新检测key

v-for遍历机制

  • 遍历的数组,原数组变化会刷新数据
  • 更改数组的单个元素,不会刷新数据, 有两个办法: 1.拿返回的新数组, 直接替换旧数组 2.this.$set()方法更新某个值
  • v-for 的遍历机制 都是操作虚拟Dom, 生成新的虚拟dom与老的虚拟dom作比较,对比结构,选择原地复用,然后渲染到页面,拥有更好的性能

虚拟Dom

  • 本质是保存节点信息, 属性和内容的一个JS对象,保存Dom的关键信息,正式Dom属性太多,操作或遍历太耗性能
  • 好处 : 在内存中层层比较变化部分, 然后就地复用相同的dom节点,再更新变化的真实的Dom

diff算法

  • 同级比较-根元素不变-属性改变更新属性
  • 如果根元素变化,删除重新建立整个DOM树
  • 根元素未变, 属性改变,DOM复用, 只更新属性

基于key来比较

  • 值唯一不重复的字符串或数字
  • 有key属性, 基于key的来比较新旧虚拟DOM, 移除key不存在元素
  • key值的使用 有id用id, 无id用索引 可以配合虚拟DOM提高更新的性能 使用数据唯一标识的作为key即可
  • 用索引做key的话,如果数据打乱了顺序,会渲染出错
  • 无key – 就地更新 / 有key – 按key比较

计算属性与侦听

计算属性

  • 基于依赖项的值进行缓存，依赖的变量不变, 都直接从缓存取结果
  <template>
  <div>
    <h1>计算属性示例</h1>
    <p>原始数据: {{ oldValue }}</p>
    <!-- newVal为20 -->
    <!-- <p>计算属性: {{ newVal }}</p> -->
    <p>计算属性值:{{ compName }}</p>
    <p>修改计算属性值: {{ val }}</p>
    <button @click="edit">修改计算属性值</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      oldValue: 10,
      val: ''
    };
  },
  //计算属性
  // computed: {
  //   //计算属性名
  //   newVal() {
  //     // 在计算属性中对原始数据进行处理
  //     return this.oldValue * 2;
  //   }
  // }

  // 计算属性完整写法
  computed: {
    // 计算属性名
    compName: {
      // get必须要有返回值,且返回值作为计算属性的值
      get() {
        return this.oldValue * 2;
      },
      // set无需返回值,接收一个参数,当修改计算属性值时触发 
      set(val) {
        // 此时val为赋予计算属性的新值
        console.log(val, 'val') // '触发了set'
      }
    }
  },
  methods: {
    edit() {
      this.compName = '触发了set'
    }
  }
};
</script>

总结

  • props里的变量本身是只读的
<!-- parent.vue -->
<template>
  <div>

    <p>父组件的数据: {{ parentData }}</p>
    <!--child为子组件  -->
    <child :childProp="parentData" />
  </div>
</template>

<script>
import child from './child.vue';
export default {
  components: {
    child
  },
  data() {
    return {
      parentData: '我是父组件中的数据'
    };
  }
};
</script>


<!-- child.vue  子组件-->
<template>
  <div>
    <h2>子组件</h2>
    <p>收到父组件的值: {{ childProp }}</p>
  </div>
</template>

<script>
export default {
  // 第一种写法  数组形式,直接接收父组件的值
  // props:['childProp']
  // 第二种形式  对象形式  指定接收的类型,是否必传,默认值,验证函数等等
  props: {
    childProp: {
      type: String,
      // required: true,
      default: '我是默认值',
      validator: function (value) {
        // 自定义验证函数  返回true和false  返回true说明验证通过,false为验证不通过  控制台报错
        return value.length <= 10;
      }
    }
  }
};
</script>

生命周期示例

export default {
  // Vue生命周期  分为四个阶段:初始化,挂载,更新,销毁        八个钩子  成对
  beforeCreate() {
    // 初始化之前的钩子,不可以获取到data,methods里的数据  表是刚创建了一个空的Vue对象,里面只有一些vue默认的事件,属性
  },
  created() {
    // 此时data,methods已经初始化好了,可以访问到里面的数据
    // 场景:可以通过ajax发起请求,获取数据渲染
  },
  beforeMount() {
    // 此时模板已近编译好了,存在内存中,但还没渲染到页面上
  },
  mounted() {
    // 此时vue实例已经创建完毕,可以获取到真实Dom
    // 场景:可以修改页面,修改Dom
  },
  beforeUpdate() {
    // 更新前执行的钩子,当执行此函数时,data里的数据是最新的,页面还是旧的,还没有同步上去
  },
  updated() {
    // 更新后的钩子,页面和data数据已经同步了,最新的
  },
  beforeDestroy() {
    // 销毁前执行的钩子,可以在销毁vue实例前做某些事,比如清除定时器
  },
  destroyed() {
    // 销毁时的钩子  完全销毁一个实例。可清理它与其它实例的连接，解绑它的全部指令及事件监听器,并不能清除DOM，仅仅销毁实例
  },
}

  actions: {

      }, 2000)

    }
  },
      getters: {
    // getters的第一个参数是state
    filterList: state => state.list.filter(item => item > 5)
  }
})

响应式原理

Object.defineProperty

  • vue2响应式原理通过Object.defineProperty劫持整个data中的属性,添加一个get和set方法,当访问或修改数据时,都会触发get或set方法来触发更新视图的逻辑
  • 缺点
    1. 此方法对数组无效,从而vue2对数组的七个变异方法进行了重写,从而达到响应式
    2. 修该或新增对象属性,或通过数组下标修改,不会触发更新,vue监听不到

观察者模式

  • 观察者模式是一种设计模式，用于建立对象之间的一对多依赖关系。当一个对象的状态发生变化时，所有依赖它的观察者都会收到通知并自动更新
  • 主要由被观察者subject,观察者observer构成 当被观察者状态发生改变时,它会遍历并通知所有观察者进行更新

发布订阅者模式

  • 主要分为发布者,订阅者和发布订阅中心 发布者不需要关心订阅者,只要将消息发布到发布订阅中心,订阅者也不需要关心发布者, 都是通过发布订阅中心来操作
